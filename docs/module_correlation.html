Module decontamination.correlation¶

This module provides different ways of computing two-point galaxy angular correlation functions. It is based on:

  • Xpol
    Tristram M. (documentation)
  • Anafast
    Górski K. M. et al. (documentation)
  • TreeCorr
    Jarvis M., Bernstein G. and Jain B. (documentation)

Angular correlation function from power spectrum¶

class decontamination.correlation.correlation_power_spectrum.Correlation_PowerSpectrum(data_lon: ndarray, data_lat: ndarray, footprint: ndarray, nside: int, nest: bool, min_sep: float, max_sep: float, n_bins: int, l_max=None, library: str = 'xpol')¶

Bases: Correlation_Abstract

Angular correlation function from power spectrum.

\[\xi(\theta)=\frac{1}{4\pi}\sum_{l=0}^{2\times\text{nside}}(2l+1)\,C_l\,P_l(\cos\theta)\]

calculate(estimator: str, random_lon: ndarray | None = None, random_lat: ndarray | None = None) → Tuple[ndarray, ndarray, ndarray]¶

Calculates the angular correlation function.

Peebles & Hauser estimator (1974):

\[\hat{\xi}=\frac{DD}{RR}-1\]

1st Landy & Szalay estimator (1993):

\[\hat{\xi}=\frac{DD-2DR-RR}{RR}\]

2nd Landy & Szalay estimator (1993):

\[\hat{\xi}=\frac{DD-DR-RD-RR}{RR}\]
Parameters:
estimatorstr

Estimator being considered (“dd”, “rr”, “dr”, “rd”, “peebles_hauser”, “landy_szalay_1”, “landy_szalay_2”).

random_lonnp.ndarray, default: None

Random catalog longitudes (in degrees). For Peebles & Hauser and Landy & Szalay estimators only.

random_latnp.ndarray, default: None

Random catalog latitudes (in degrees). For Peebles & Hauser and Landy & Szalay estimators only.

Returns:
Tuple[np.ndarray, np.ndarray]

The bin of angles \(\theta\) (in arcmins), the angular correlations \(\xi(\theta)\) and the angular correlation errors \(\sigma_\xi(\theta)\).

Maximum separation (in degrees).

Number of angular bins.

Angular correlation function from pair counting¶

class decontamination.correlation.correlation_pair_count.Correlation_PairCount(data_lon: ndarray, data_lat: ndarray, min_sep: float, max_sep: float, n_bins: int, bin_slop: float | None = None, n_threads: int | None = None, footprint: ndarray | None = None, coverage: ndarray | None = None, nside: bool | None = None, nest: bool = True)¶

Bases: Correlation_Abstract

Galaxy angular correlation function using the TreeCorr library. Supports pair counting (≡ NN correlations) or a scalar field approach (≡ KK correlations) if a footprint is provided.

Parameters:
data_lonnp.ndarray

Galaxy catalog longitudes (in degrees).

data_latnp.ndarray

Galaxy catalog latitudes (in degrees).

min_sepfloat

Minimum galaxy separation being considered (in arcmins).

max_sepfloat

Maximum galaxy separation being considered (in arcmins).

n_binsint

Number of angular bins.

bin_slopOptional[float]

Precision parameter for binning (see TreeCorr documentation).

n_threadsOptional[int], default: None ≡ the number of cpu cores

How many OpenMP threads to use during the calculation.

footprintOptional[np.ndarray]

HEALPix indices of the region where correlation must be calculated (KK correlations only).

coverageOptional[np.ndarray]

Observed sky fraction for each of the aforementioned HEALPix pixels (KK correlations only).

nsideOptional[np.ndarray]

The HEALPix nside parameter (KK correlations only).

nestbool, default: True

If True, assumes NESTED pixel ordering, otherwise, RING pixel ordering (KK correlations only).
